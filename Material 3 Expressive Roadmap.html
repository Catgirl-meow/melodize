01 Shape morphing on the mini player P0~4h

The single strongest M3E idea and the one that fits Melodize best. Shapes are no longer static — they morph on state change.

What

When the mini player is playing, its 16 px rounded rectangle should breathe into a softer, slightly-asymmetric shape (e.g. 22 px leading, 18 px trailing) synced to beat detection or the scrub progress. On pause, it returns to uniform 16 px. The same treatment applies to the album thumbnail inside it — square at rest, squircle while playing.

Why for Melodize

You already re-tint the whole dock from artwork. Shape-morphing gives the app a second axis of personality that most Subsonic clients can’t replicate. It’s a low-risk visual upgrade because it’s ignorable — static shapes still look correct.

edits: lib/features/player/widgets/floating_mini_player.dart
In lib/features/player/widgets/floating_mini_player.dart, replace the static ClipRRect(borderRadius: 16) with an AnimatedContainer whose borderRadius responds to AudioService.playbackState. When playing: BorderRadius.only(topLeft: 22, bottomLeft: 22, topRight: 18, bottomRight: 18). When paused: uniform BorderRadius.circular(16). Use Curves.easeInOutCubicEmphasized (400 ms). Do the same to the leading album thumbnail — 12 px when paused, 18 px when playing. Keep the existing blur+tint layer untouched. Add a RepaintBoundary around the AnimatedContainer so we don’t dirty the surrounding dock.
Acceptance
  • Play/pause toggle visibly changes the mini player’s corner asymmetry.
  • No new jank on profile — repaint cost stays within 1.5 ms/frame on a Pixel 6.
  • Works in both light and dark themes without color shifts.

02 M3E top app bars on Home, Library, Settings P0~3h

Top-level destinations should use large or medium app bars that collapse on scroll. All three currently use the small variant.

What

  • Home → SliverAppBar.large, title "Home" or user greeting, collapses from 152 → 64 px.
  • Library → SliverAppBar.medium (112 px), tabs pin below.
  • Settings → SliverAppBar.large, matches Android 14+ Settings idiom.
edits: home/home_screen.dart, library/library_screen.dart, settings/settings_screen.dart
Convert home_screen, library_screen, and settings_screen to use CustomScrollView with SliverAppBar. For Home and Settings use SliverAppBar.large; for Library use SliverAppBar.medium with a TabBar pinned underneath via SliverPersistentHeader. The large app bar should show a 36 sp title (M3E "headlineMedium" equivalent) when expanded and collapse to 22 sp. Keep your existing body content as a list of slivers. Set scrolledUnderElevation: 0 and surfaceTintColor: Theme.of(context).colorScheme.surfaceContainer so the bar tints correctly on scroll without a shadow. Do not add back buttons — these are top-level destinations.
Acceptance
  • Each screen’s app bar expands on initial load and collapses on scroll without jitter.
  • Title size animates smoothly between expanded/collapsed states.
  • Library’s tab bar stays pinned below the collapsed app bar.

03 Grouped list tiles in Settings P0~2h

M3E introduces grouped settings with shared outer radius and tight inner corners — the Android 15 Settings look.

What

In each section (Appearance, Playback, Downloads, Server, Deezer, Companion), adjacent tiles should share a 24 px outer radius on the group’s top and bottom, with 4 px inner radius on all touching edges between tiles. Single-item groups get full 24 px on all corners.

new: lib/widgets/grouped_list_tile.dart · edits: settings/settings_screen.dart
Create lib/widgets/grouped_list_tile.dart exporting a GroupedListTile widget and a GroupedSection wrapper. GroupedSection takes a List<Widget> and computes each child’s borderRadius based on position: first child = BorderRadius.vertical(top: Radius.circular(24)) + bottom corners 4px; middle = 4px all corners; last = bottom 24px + top 4px; only-child = 24px all corners. Each tile gets a colorScheme.surfaceContainerHigh background with 2 px vertical gaps between tiles in the group (use a SizedBox with height 2). Refactor settings_screen.dart to wrap each existing section’s tiles in GroupedSection. Keep the existing TextStyle, leading icons, and trailing switches intact.
Acceptance
  • Settings sections visually chunk as pills instead of flat rows.
  • 4 px gutters between tiles in the same group; 24 px section breathing room.
  • Works in light mode without losing contrast.

04 Expressive loading and button feedback P1~3h

Replace CircularProgressIndicators with M3E’s wavy LinearProgressIndicator and adopt the new ProgressIndicator shape morphs.

What

  • The 4-dot loading indicator in your Now Playing lyrics panel → LinearProgressIndicator(year: 2023, waveAmplitude: 4) with accent color.
  • Play/pause FAB on Now Playing → M3E’s shape-morph button (rounded square ⇄ circle on state change).
  • Download progress in the Library’s Downloaded view → wavy horizontal bar with animated stops.
edits: lib/features/player/now_playing_screen.dart, lib/widgets/wavy_progress.dart
Upgrade Flutter to ^3.27 (channel stable) if not already, then use LinearProgressIndicator’s new waveAmplitude and waveLength parameters. Create lib/widgets/wavy_progress.dart that wraps LinearProgressIndicator with defaults: waveAmplitude: 3.0, waveLength: 32.0, color: Theme.of(context).colorScheme.primary, backgroundColor: colorScheme.primaryContainer.withOpacity(0.3). Replace the existing CircularProgressIndicator usages in lyrics loading states. For the play/pause FAB in now_playing_screen, wrap the FloatingActionButton’s shape in AnimatedContainer that morphs between RoundedRectangleBorder(22) when paused and StadiumBorder() (circle) when playing — 300ms easeInOutCubicEmphasized.
Acceptance
  • Lyrics loading shows a horizontal wavy bar animating left-to-right.
  • Play/pause FAB morphs shape smoothly on each tap.
  • Shuffle/repeat toggles use the new filled-icon-button style (IconButton.filled).

05 Re-seed the color scheme with DynamicSchemeVariant.expressive P1~1h

Flutter 3.27 exposes M3E’s new tonal-palette variants. Swap tonalSpot for expressive or vibrant.

What

Your ColorSchemeBuilder currently calls ColorScheme.fromSeed(seedColor: artworkSeed) which defaults to DynamicSchemeVariant.tonalSpot. M3E ships expressive (playful, higher chroma) and vibrant (maximum saturation). Melodize’s artwork-driven tinting will feel dramatically richer under expressive without losing legibility.

edits: lib/theme/app_theme.dart or wherever ColorScheme.fromSeed is called
Find every call to ColorScheme.fromSeed in the codebase. Add the parameter dynamicSchemeVariant: DynamicSchemeVariant.expressive to each call, after the seedColor arg. If the call is inside the artwork-palette path (PaletteGenerator result → seed), use DynamicSchemeVariant.vibrant instead for stronger artwork-driven personality. Also bump contrastLevel: 0.3 on both light and dark scheme construction to satisfy WCAG AA on the secondary-container backgrounds that currently fail (see design review §01 note 4).
Acceptance
  • Home feed cards visibly pop more on colorful artwork (e.g. Ado, Kenshi Yonezu albums).
  • Text on primaryContainer passes WCAG AA everywhere (≥4.5:1).
  • No regressions in light mode on pale artwork.

06 Emphasized motion everywhere P1~4h

M3E’s motion system uses emphasized and standard families — not the old easeInOut.

What

Create a MotionTokens class that matches M3E spec: emphasized (400 ms, cubicEmphasized), emphasizedAccelerate (200 ms, cubicEmphasizedAccelerate), standard (300 ms, easeInOutCubic). Replace hard-coded Duration(milliseconds: X) + Curves.easeInOut calls across the app.

new: lib/theme/motion_tokens.dart · edits: every animated widget (~22 files)
Create lib/theme/motion_tokens.dart exporting a static class MotionTokens with these constants: emphasizedDuration = Duration(milliseconds: 400); standardDuration = Duration(milliseconds: 300); emphasizedCurve = Cubic(0.2, 0.0, 0.0, 1.0); emphasizedAccelerate = Cubic(0.3, 0.0, 0.8, 0.15); emphasizedDecelerate = Cubic(0.05, 0.7, 0.1, 1.0); standardCurve = Curves.easeInOutCubic. Then do a codebase-wide refactor: replace every Duration(milliseconds: 200/300/400) paired with Curves.easeInOut in AnimatedContainer, AnimatedOpacity, TweenAnimationBuilder, PageRouteBuilder, and AnimationController with the appropriate MotionTokens constant. Use emphasizedDuration for state changes (play/pause, sheet open), standardDuration for incidental things (hover, ripple). Report each file you edit with the before/after lines.
Acceptance
  • All major state transitions feel snappier and more deliberate.
  • Now Playing → Lyrics pager page swipes use emphasizedDecelerate.
  • No animation takes longer than 400 ms end-to-end.

07 Typography: adopt M3E displayLargeEmphasized P2~2h

M3E’s 2025 type scale adds an "emphasized" axis for every text style — heavier weight, tighter tracking.

What

Use the emphasized styles on: Now Playing song title (currently 22 sp, 700), section headers in Home (currently 14 sp, 600), and the Library tab labels. M3E spec: titleLargeEmphasized = 22 sp, weight 700, tracking -0.2. labelLargeEmphasized = 14 sp, weight 700, tracking 0.

edits: lib/theme/app_theme.dart
Extend the app TextTheme to include the M3E emphasized variants. Add these custom TextStyles as extensions on TextTheme via ThemeExtension: displayLargeEmphasized, titleLargeEmphasized, labelLargeEmphasized. Values per M3E spec: titleLargeEmphasized = TextStyle(fontSize: 22, fontWeight: FontWeight.w700, letterSpacing: -0.2, height: 1.27); labelLargeEmphasized = TextStyle(fontSize: 14, fontWeight: FontWeight.w700, letterSpacing: 0, height: 1.43). Apply titleLargeEmphasized to the now playing song title and labelLargeEmphasized to Home’s section titles ("Made for you", "Recently Played", etc). Keep the font family the same (Google Sans / platform default).
Acceptance
  • Now Playing title visibly heavier and tighter than before.
  • Home section headers no longer feel undersized.
  • No text clipping on narrow devices.

08 Haptics alongside motion P2~1h

M3E couples shape/motion changes with subtle haptic feedback.

What

Add HapticFeedback.selectionClick() on: dock tab switches, play/pause, queue reorder, scrub release. Use HapticFeedback.lightImpact() on the sheet-dismiss over-scroll in Now Playing.

edits: floating_dock.dart, now_playing_screen.dart, queue_sheet.dart
Add HapticFeedback calls (from flutter/services.dart) to key interaction points. In floating_dock.dart’s onDestinationSelected, call HapticFeedback.selectionClick() before the setState. In now_playing_screen.dart’s play/pause button onTap, call HapticFeedback.lightImpact() alongside the audio command. On the scrub bar’s onChangeEnd, call HapticFeedback.selectionClick(). On the queue sheet’s ReorderableListView, HapticFeedback.mediumImpact() in onReorderStart and selectionClick in onReorderEnd. Gate all haptic calls behind a user setting (add a new SharedPreferences key "haptics_enabled" defaulting to true) and expose it as a switch tile under Settings → Appearance.
Acceptance
  • Every primary interaction has a subtle tactile response on devices that support it.
  • Users can disable haptics from Settings.
  • No haptics fire on iOS web/desktop targets (guarded by platform check).

⌘ Suggested sequencing

Ship in this order for maximum perceived polish at minimum risk.

Week 1 (P0): 05 re-seed (1h) → 02 app bars (3h) → 03 grouped settings (2h) → 01 shape morph (4h). Ship as 0.2.0. Visible impact: richer colors, proper hierarchy, the signature mini-player breath.

Week 2 (P1): 06 motion tokens (4h) → 04 wavy progress + FAB morph (3h) → 05 re-seed on vibrant variant (revisit if too saturated). Ship as 0.2.1.

Later (P2): 07 type emphasized + 08 haptics. These are small and can land alongside any feature work.
